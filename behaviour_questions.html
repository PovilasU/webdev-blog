React Fundamentals

Q1: Can you explain the difference between functional and class components in React? When would you use one over the other?
Class components are ES6 classes that extend React.Component and use lifecycle methods like componentDidMount and componentDidUpdate to manage state and side effects. Functional components are simpler functions that return JSX. With the introduction of hooks, functional components can now manage state and side effects using hooks like useState and useEffect. Today, functional components are preferred because they are more concise, easier to read, and hooks provide powerful capabilities for managing state and lifecycle events without the verbosity of classes.
Q2: What does the useEffect hook do, and can you give an example of how and when you'd use it?
The useEffect hook lets you perform side effects in functional components, such as fetching data, setting up subscriptions, or manually changing the DOM. It runs after the render and can be configured to run only once on mount or whenever specific dependencies change by using the dependency array. Example: Fetch data when the component mounts: 
useEffect(() => {
  fetchData();
}, []); // empty dependency array means this runs once on mount
Q3: What is the difference between useState and useRef, and when would you use useRef instead of useState?
useState is a hook that manages state in functional components and triggers a re-render whenever the state changes. useRef, on the other hand, returns a mutable ref object whose .current property persists across renders but updating it does not trigger a re-render. You use useRef to: - Access or store DOM elements (e.g., focusing an input). - Store mutable values that don’t affect rendering, like timers or previous values. Example: 
const inputRef = useRef(null);

useEffect(() => {
  inputRef.current.focus();
}, []);

return <input ref={inputRef} />;
Q4: Say you have a list of items that the user can add to or remove from. How would you manage this list in React? What hooks would you use?
I would use the useState hook to manage the list as an array. To update the list immutably, I would use the spread operator. Example: 
const [items, setItems] = useState([]);

// Add an item
const addItem = (newItem) => {
  setItems(prevItems => [...prevItems, newItem]);
};

// Remove an item by id
const removeItem = (id) => {
  setItems(prevItems => prevItems.filter(item => item.id !== id));
};
Q4.1: Would you consider using useReducer instead of useState here? Why or why not?
useReducer is useful when state logic is complex, such as when you have multiple related state updates or different types of actions that affect the state. It provides a more predictable state management pattern similar to Redux reducers. For simple lists, useState is sufficient. But if the list logic grows more complex, with many actions (add, remove, update, reset), I would consider useReducer to better organize the state updates. Example reducer: 
const reducer = (state, action) => {
  switch (action.type) {
    case 'add':
      return [...state, action.payload];
    case 'remove':
      return state.filter(item => item.id !== action.payload.id);
    default:
      return state;
  }
};

const [items, dispatch] = useReducer(reducer, []);

Q5: Can you describe a challenge you faced in a React project and how you solved it?
In a previous role at Allente, I was tasked with migrating a Blazor ASP.NET application to React.js. The migration had to be gradual since the React code was injected into ASP.NET .cshtml pages, making it a hybrid app rather than a full React SPA. I carefully embedded React components within the existing views using ReactDOM.render and passed necessary data via model properties or global JavaScript objects. A major challenge was ensuring smooth data flow and maintaining consistent state between the old and new components. I addressed this by writing integration tests, performing manual verifications, and collaborating closely with QA. This approach minimized disruption and allowed a successful incremental migration to React.

        import React, { useState } from 'react';

        function TodoList() {
        const [task, setTask] = useState('');
        const [tasks, setTasks] = useState([]);

        const handleAddTask = () => {
            if (task.trim() === '') return; // avoid empty tasks
            setTasks([...tasks, task]);
            setTask('');
        };

        return (
            <div>
            <h2>Todo List</h2>
            <input 
                type="text" 
                value={task} 
                onChange={(e) => setTask(e.target.value)} 
                placeholder="Enter a task" 
            />
            <button onClick={handleAddTask}>Add</button>

            <ul>
                {tasks.map((t, index) => (
                <li key={index}>{t}</li>
                ))}
            </ul>
            </div>
        );
        }

        export default TodoList;

import React, { useState } from "react";

function Child({ sendDataToParent }) {
  return (
    <button onClick={() => sendDataToParent("Hello from Child!")}>
      Click me to send data to parent
    </button>
  );
}

export default function App() {
  const [message, setMessage] = useState("");

  // Function to receive data from child
  function handleDataFromChild(data) {
    setMessage(data);
  }

  return (
    <div className="App">
      <h1>Hello CodeSandbox</h1>
      <h2>Start editing to see some magic happen!</h2>

      <Child sendDataToParent={handleDataFromChild} />

      <p>Message from child: {message}</p>
    </div>
  );
}
