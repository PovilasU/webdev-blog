Question Answer
1. What is React? A JavaScript library for building UI with a component-based approach.
2. What is JSX? Syntax extension that looks like HTML in JavaScript.
3. Props vs State? Props are read-only and passed from parent; state is internal and mutable.
4. useState Adds state to a functional component.
5. useEffect Handles side effects like data fetching.
6. When does useEffect run? After render or when dependencies change.
7. What is the virtual DOM? A lightweight copy of the real DOM for performance optimization.
8. Functional components? Components written as functions instead of classes.
9. Controlled components? Inputs that get their value from state.
10. Key in React lists? Unique identifier used to optimize rendering of lists.
11. Passing data to child? Use props.
12. Child to parent? Call function passed as prop.
13. Conditional rendering? Show components based on conditions using if/ternary.
14. onClick event? Example: <button onClick={handleClick}>
15. Render a list? Use .map() and key for each item.
16. React Router? Used for client-side routing.
17. Fetching data? Use fetch in useEffect.
18. Update state directly? No. Use setState or state updater function.
19. What is a hook? Function that lets you use React features in function components.
20. useRef? Access DOM nodes or persist values across renders.
